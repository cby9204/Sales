영업활동 monday.com 가이드

리드 생성부터 영업활동 관리, 거래 진행, 계약 완료 및 종료 처리까지 영업 프로세스 전반의 실무 기준을 정리한 문서입니다. 각 단계별 업무 기준과 필수 입력 사항, 운영 원칙을 정리하였으니 실무 진행 시 참고 부탁드립니다.

01

영업활동 그라운드룰

monday.com을 활용한 영업활동의 기본 운영 원칙입니다. 모든 영업 담당자는 아래 8가지 원칙을 숙지하고 따라주세요.

No. 원칙 설명
1Single Source of Truth모든 영업 정보는 monday.com CRM에 기록합니다. 개인 메모, 엑셀, 메신저 등에만 저장된 정보는 공식 영업 데이터로 인정되지 않습니다.
2거래 → 종료 단계 준수모든 영업 기회는 반드시 거래 → 종료 순서로 흘러야 합니다. 단계를 건너뛰거나 임의로 보드를 옮기지 않습니다.
3실시간 상태 업데이트 (T+1)영업활동(미팅, 제안, 견적, 협의 등)이 발생하면 당일 또는 익일 오전까지 거래 보드 상태값을 최신화합니다. 상태값 지연은 파이프라인 왜곡을 야기합니다.
4필수값 누락 금지각 보드별 필수 입력값(담당자, 기업명, 금액, 종료사유 등)은 단계 전환 전 반드시 입력합니다. 누락 시 거래/종료 보드 전환이 제한될 수 있습니다.
5활동 이력은 업데이트 섹션에영업활동 과정의 모든 커뮤니케이션은 해당 아이템의 업데이트(코멘트) 섹션에 기록합니다. 다음 담당자가 이력만 봐도 맥락 파악이 가능해야 합니다.
6수주/실패 즉시 종료 처리결과가 확정된 건은 영업일 기준 24시간 이내 종료 보드로 이동합니다. 실패 건도 사유를 반드시 남겨 향후 재영업 자산으로 활용합니다.
7공유 보기 임의 변경 금지기본 보기 고정, 필터 저장, 컬럼 순서 변경은 전체 사용자에게 반영되므로 절대 사용하지 않습니다. 개인용 설정만 자유롭게 조정합니다.
8담당자 변경 시 인수인계담당자 변경 시 기존 담당자는 업데이트 섹션에 진행 현황·다음 액션·고객 특이사항을 정리한 인수인계 코멘트를 남깁니다.

03

전체 영업 프로세스 흐름

리드(Lead) → 거래(Deal) → 종료(Closed) 의 3단계 보드 구조로 운영됩니다. 모든 영업 기회는 리드 보드에서 시작하며, 단계 전환은 정해진 기준에 따라서만 진행됩니다.

STEP 01
리드 보드 Lead
신규 유입 및 초기 검토 단계. 영업 대상 기업/기회의 최초 등록과 검수, 우선순위 판단을 수행.
주 담당 · 관리자 / 운영 담당 전환 · 담당자 배정 완료 → 거래 보드
STEP 02
거래 보드 Deal
실제 영업활동 및 매출화 진행 단계. 미팅·제안·견적·협의 등 모든 영업활동이 실행되는 핵심 보드.
주 담당 · 영업 담당자 전환 · 수주/실패 확정 → 종료 보드
STEP 03
종료 보드 Closed
결과 확정 및 이력 관리 단계. 수주/실패 결과 기록, 실적 집계, 재영업 대상 발굴 자료로 활용.
주 담당 · 영업 담당자 + 관리자 전환 · 종착지 (재영업 시 신규 리드)
04

리드 보드 Lead Board

4-1목적과 운영 원칙

  • 모든 영업 기회의 시작점. 신규/재영업/내부발굴/외부채널 인입 등 발생 경로와 무관하게 모든 대상은 우선 리드 보드에 생성됩니다.
  • 담당 영업자가 직접 영업활동을 수행하는 공간이 아닌, 배정 전 대기 및 검토 단계의 성격을 가집니다.
  • 리드 보드 등록 → 영업 시작. 담당자 배정 후 거래 보드로 이동된 시점부터 실제 영업활동이 시작됩니다.

05

거래 보드 Deal Board

5-1목적과 운영 원칙

  • 실제 영업활동이 이루어지는 핵심 보드. 현재 매출 가능성이 살아있는 모든 파이프라인이 모이는 공간입니다.
  • 포함되는 리드: ① 새롭게 배정되어 영업을 시작해야 하는 신규 리드 ② 제안·견적·협의 등 영업활동이 진행 중인 기존 리드.
  • 거래 보드 데이터는 개인 성과·팀 파이프라인·예상 매출 Forecast·전환율 분석의 기준이 되므로 정확한 상태 업데이트와 누락 없는 데이터 입력이 필수입니다.

6-1목적과 운영 원칙

  • 딜이 최종적으로 마무리된 건을 관리하는 보드. '수주(계약 완료)' 또는 '실패(보류·경쟁사 선정·예산 미확보·미진행 등)' 두 가지 결과로 확정 시 이동.
  • 단순 보관이 아닌 결과 기록·분석·재활용을 위한 보드. 월별/담당자별 실적 집계, 수주율·실패사유 분석, 재영업 대상 발굴, 계약 이력 관리, 고객 히스토리 추적이 가능.
  • 수주 건은 매출 집계와 직접 연결되므로 계약금액·상품명·시작일·종료일 등 필수값을 정확히 입력해야 합니다.
  • 실패 건은 단순 종료 처리하지 말고 사유를 반드시 남겨 향후 재접촉 전략 수립 자산으로 활용합니다.

❗ 자주 하는 실수 (Common Mistakes)

실무에서 반복적으로 발생하는 입력 실수들입니다. 등록 전 한 번 더 체크해주세요.

Q. 1개의 기업이 동시에 여러 상품을 구매하는 경우 어떻게 등록하나요? ex) 광고 + 공고 구매
→ 각 캠페인 ID 및 플랜 ID 별로 하나씩 거래 완료가 되어 있어야 합니다. 합쳐서 등록하지 마세요.
❌ 잘못된 예
캠페인 ID: 4567, 7896 (한 거래에 합쳐 입력)
계약금액: 1,000,000 (합계로 입력)
거래 건수: 1개 거래에 통합
✅ 올바른 예
거래 ①: 4567 / 500,000
거래 ②: 7896 / 500,000
거래 건수: 2개 거래로 각각 등록
⚠️
캠페인 ID 다중 입력 + 합계 금액 입력은 실적 집계 오류의 가장 큰 원인입니다.
Q. 자소설페이는 어떻게 등록하나요?
→ '충전 시점'과 '사용 시점'을 분리해서 각각 등록합니다.
구분 캠페인 ID 페이 계약 여부 상품금액 입력
충전 (예: 1억 충전)0O충전 금액(1억)을 상품금액 필드에 입력
사용 (예: 500만원 사용)실제 사용한 캠페인/플랜 IDO실 사용 금액(500만원) 입력
💡
자소설페이는 충전과 사용이 별개로 집행되므로 반드시 분리해서 등록합니다. 두 케이스 모두 '페이 계약 여부 = O' 표시 필수.
Q. 캠페인 ID 필드인데 공고 진행 건은 어떻게 적나요?
→ 필드명은 '캠페인 ID'로 되어 있지만, 공고 진행 건은 '플랜 ID'를 작성하시면 됩니다.
